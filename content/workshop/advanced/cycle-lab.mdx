# 실습: 자동 개선 사이클

<Callout type="warning">
**이 세션의 핵심**

autoresearch에서 사람은 점수를 직접 매기지 않습니다.
AI가 알아서 실험하고, 채점하고, 개선하고, 반복합니다.
사람은 **방향만 조정**합니다.

오늘 여러분도 똑같이 합니다.
Gemini Canvas 앱이 자동으로 프롬프트를 개선하는 것을 **지켜보면서, 방향만 조정**합니다.
</Callout>

<Steps>

## Step 1: prompt-workshop fork + 본인 맥락 수정

<Timer minutes={15} />

### 저장소 fork

1. [prompt-workshop](https://github.com/effectiveai-io/prompt-workshop) 접속
2. 우측 상단 **Fork** 클릭 → 본인 GitHub 계정으로 fork

### 본인 직군에 맞는 파일 선택

| 직군 | 파일 |
|------|------|
| BE/FE 개발 | `prompts/code-review.md` |
| QA | `prompts/test-case.md` |
| PM/기획 | `prompts/spec-review.md` |
| 공통 | `prompts/document-draft.md` |

### GitHub 웹에서 수정

1. fork한 저장소에서 선택한 파일 클릭
2. 연필 아이콘(Edit) 클릭
3. 본인 팀 맥락에 맞게 수정:
   - 팀명, 서비스명을 본인 것으로
   - "팀 컨벤션" 섹션에 실제 규칙 추가
   - "도메인 맥락" 섹션에 우리 서비스 특수 규칙 추가
4. Commit changes

<Callout type="info">
지금 실제 팀 코드가 없으니 완벽하지 않아도 됩니다.
돌아가서 실제 맥락을 추가하면 진짜 자산이 됩니다.
</Callout>

## Step 2: 자동 개선 앱 만들기 (Gemini Canvas)

<Timer minutes={10} />

Gemini Canvas로 **프롬프트를 자동으로 개선하고 점수 변화를 시각화하는 앱**을 만듭니다.
이 앱이 autoresearch의 실험 루프를 대신합니다.

Gemini에서 Canvas 모드를 켜고 아래 프롬프트를 입력하세요:

<Tabs items={['자동 개선 앱 프롬프트']}>
<Tabs.Tab>

```
Gemini Canvas로 "프롬프트 자동 개선기" 웹앱을 만들어줘.
autoresearch처럼 AI가 자동으로 프롬프트를 개선하는 루프를 돌리는 앱이야.

## 화면 구성

### 상단: 입력 영역
- "프롬프트" 텍스트 영역 (여러 줄 입력 가능, 기본값 포함)
- "평가 기준" 텍스트 영역 (10개 항목, 각 1~5점, 만점 50점)
- "테스트 시나리오" 텍스트 영역 (프롬프트를 테스트할 샘플 입력)
- "사이클 수" 선택 (1~5, 기본값 3)
- "자동 개선 시작" 버튼

### 하단: 결과 영역
- 사이클별 총점 바 차트 (v0, v1, v2... 50점 만점 기준으로 점수가 올라가는 것을 시각화)
- 각 사이클의 상세 결과 (접었다 펼 수 있는 아코디언):
  - 사용한 프롬프트
  - 생성된 결과물
  - 10개 항목별 점수표 (1~5점, 이전 대비 +/- 표시)
  - 이번에 개선한 것 / 다음에 개선할 것
- 최종 개선된 프롬프트 복사 버튼

## 동작 방식

"자동 개선 시작" 버튼을 누르면:
1. 입력된 프롬프트를 사용해서 테스트 시나리오에 대한 결과물을 생성
2. 평가 기준으로 결과물을 채점
3. 점수가 낮은 항목을 분석하고 프롬프트를 개선
4. 개선된 프롬프트로 1번부터 다시 반복
5. 설정된 사이클 수만큼 반복

각 사이클이 완료될 때마다 바 차트가 실시간으로 업데이트됨.

## 디자인
- 한국어 UI
- 다크 모드
- 바 차트: 점수 낮으면 회색, 중간이면 파란색, 높으면 초록색
- 진행 중인 사이클은 로딩 애니메이션 표시
- 깔끔하고 전문적인 느낌

## 기본값

프롬프트 기본값:
"아래 코드를 리뷰해주세요."

평가 기준 기본값:
"10개 항목, 각 1~5점 (만점 50점)
1. 네이밍 컨벤션 (1~5)
2. 레이어 분리 (1~5)
3. 보안 취약점 (1~5)
4. 금액/데이터 정확성 (1~5)
5. 동시성/경쟁 조건 (1~5)
6. 에러 처리 (1~5)
7. 테스트 용이성 (1~5)
8. 리뷰 심각도 분류 (1~5)
9. 코드 예시 품질 (1~5)
10. 도메인 우선순위 (1~5)

1점: 언급 없음, 2점: 언급만, 3점: 일부 반영, 4점: 구체적, 5점: 전문가 수준"

테스트 시나리오 기본값:
(간단한 Java 코드 스니펫)
```

## Step 3: 자동 개선 루프 돌리기

<Timer minutes={20} />

<Callout type="warning">
**여기가 autoresearch 본질을 경험하는 순간입니다.**

사람은 버튼만 누릅니다. AI가 알아서 실험하고, 채점하고, 개선합니다.
</Callout>

### 첫 실행: fork한 프롬프트로

1. Step 1에서 수정한 프롬프트(`prompts/code-review.md` 등)를 Canvas 앱의 "프롬프트" 영역에 붙여넣기
2. fork한 저장소의 `evaluation/code-review.md` 내용을 "평가 기준" 영역에 붙여넣기
3. fork한 저장소의 `samples/sample-code.md` 내용을 "테스트 시나리오" 영역에 붙여넣기
4. **"자동 개선 시작" 클릭**

→ AI가 자동으로 사이클을 돌면서 바 차트가 올라가는 것을 지켜봅니다.

### 관찰하기

자동 루프가 돌아가는 동안:
- **바 차트**: 총점이 사이클마다 올라가는지 확인 (50점 만점 — 보통 15점대에서 시작, 30점대까지 올라가면 좋은 결과)
- **항목별 점수**: 10개 항목 중 어떤 것이 올라가고, 어떤 것이 1~2점에 머물러 있는지
- **AI의 개선 방향**: 각 사이클에서 AI가 뭘 바꿨는지 아코디언을 열어 확인
- **정체 구간**: 총점이 더 이상 안 올라가는 순간이 오면 — 그게 방향 조정이 필요한 신호

### 방향 조정 후 재실행

첫 실행 결과를 보고, 원하는 방향이 아니면:

1. AI가 만든 **최종 프롬프트**를 복사
2. 그 위에 본인이 원하는 방향을 추가 (예: "보안 규칙을 더 강화해줘")
3. 다시 **"자동 개선 시작"** 클릭

> **이것이 program.md를 다듬는 경험입니다.**
> 첫 실행의 최종 프롬프트가 다음 실행의 v0이 됩니다.
> 사이클을 거듭할수록 프롬프트가 더 정교해집니다.

<Callout type="info">
시간이 남으면 2~3번 더 돌려보세요.
매번 방향을 조금씩 바꿔가며 돌리면, autoresearch가 program.md를 다듬는 것과 같은 경험이 됩니다.
</Callout>

## Step 4: 결과 확인 + 저장

<Timer minutes={10} />

### 전체 흐름 확인

- 첫 v0에서 마지막 사이클까지 총점이 얼마나 올랐는지
- **가장 큰 점수 상승이 있었던 사이클**: AI가 뭘 바꿨는지가 핵심 인사이트
- **여전히 낮은 항목**: 돌아가서 실제 맥락을 추가하면 올릴 수 있는 부분

### 최종 프롬프트 저장

Canvas 앱에서 최종 프롬프트를 복사합니다:
- GitHub fork에 커밋하거나 메모장에 저장
- 이것이 **AI가 자동으로 만들어준, 검증된 업무 프롬프트**입니다

</Steps>

<Callout type="info">
autoresearch는 밤새 100번 돌립니다.
여러분은 몇 번 돌렸을 뿐인데도 프롬프트 품질이 올라갔습니다.
돌아가서 실제 코드, 실제 기획서로 바꿔 넣고 다시 돌리면 어떻게 될까요?
</Callout>
