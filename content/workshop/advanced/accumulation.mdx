# 반복에서 축적으로

> 오늘 경험한 사이클을 "한 번의 워크숍"으로 끝내지 않고,
> 팀과 조직으로 확장하는 그림을 봅니다.

## 축적의 3레벨

```
개인 → 도구:   오늘 만든 지침서 = 내 반복 업무의 첫 번째 자동화
팀 → 자산:     지침서를 팀에 공유하고 피드백 받으면 = 팀 표준
조직 → 문화:   축적된 자산 위에서 누구나 쓸 수 있으면 = 문화 전환
```

## 레벨별 우아한형제들 사례

### 개인 → 도구

> [AI와 함께하는 테스트 자동화: 플러그인 개발기](https://techblog.woowahan.com/24568/) (강규한, 2025.12)

매번 같은 컨벤션 설명을 반복하던 개발자가, 플러그인으로 구조화해서 70% 시간을 단축했습니다.

오늘 여러분이 한 것과 같습니다. <strong>반복을 도구로 만들었습니다.</strong>

### 팀 → 자산

> [흩어져 있는 AI 자산, 'MCP stdio'로 헤쳐모여!](https://techblog.woowahan.com/25986/) (조현석, 2026.03)

개인 메모장에 흩어진 프롬프트·규칙을 npm 패키지로 중앙화했습니다.
`npm install` 한 번으로 팀 전원이 같은 수준에서 시작합니다.

<strong>공유가 아니라 설치로 전파됩니다.</strong>

### 조직 → 문화

> [AI로 바뀐 건 업무가 아니라 사람이었습니다](https://techblog.woowahan.com/26034/) (김사랑, 2026.03)

사내 AI 교육 프로그램 "AI Dive Deep"에서:
- 마케터가 데이터 취합 150분 → 25분으로 단축
- PM이 MCP로 코드 정책 직접 확인 (코드를 읽지 않아도)
- 디자이너가 Cursor AI로 피그마 플러그인 직접 제작

개발자가 만든 도구와 자산 위에서, 비개발 직군도 AI를 실무에 적용하게 됐습니다.

<Callout type="info">
조직 문화는 선언으로 만들어지지 않습니다.
<strong>개인이 도구를 만들고, 팀이 자산을 쌓고, 그 위에서 누구나 쓸 수 있을 때</strong> 비로소 문화가 됩니다.
</Callout>

## 오늘 만든 것은 어디에 있는가?

| 오늘 한 것 | 현재 위치 | 다음 스텝 |
|-----------|----------|----------|
| 지침서 v2 완성 | 개인 → 도구 | 팀 위키에 올려 피드백 받기 |
| 교환 테스트 통과 | 개인 → 도구 경계 | 팀원 3명 이상이 써보고 개선하기 |
| 팀 표준으로 채택 | 팀 → 자산 | ChatGPT 팀 워크스페이스에 반영 |

## 도구 이름은 달라도 구조는 같다

| 도구 | 지침서가 들어가는 곳 |
|------|---------------------|
| ChatGPT | 커스텀 지침 (Settings → Customize) |
| Claude | 프로젝트 지침 (Project Instructions) |
| Claude Code / Antigravity | `CLAUDE.md` / `AGENTS.md` |
| Cursor | `.cursorrules` |
| GitHub Copilot | `copilot-instructions.md` |
| Gemini | Gems |

오늘 만든 지침서는 ChatGPT 커스텀 지침에 넣었지만, 내용은 어떤 도구로든 옮길 수 있습니다.
지침서의 본질은 도구가 아니라 <strong>여러분의 업무 지식</strong>이니까요.

## 회고 (10분)

**개인 회고** — 3문항:
1. 3사이클 중 점수가 가장 크게 오른 순간은? 무엇을 추가했을 때?
2. 교환 테스트에서 발견한 것은?
3. 월요일에 바로 할 수 있는 것 하나는?

소그룹에서 3번 답변만 한 문장으로 공유합니다.

## 액션플랜

| 타이밍 | 액션 |
|--------|------|
| 이번 주 | 오늘 만든 지침서를 실제 업무에서 3일간 써보고, 매일 한 줄씩 개선하기 |
| 다음 주 | 지침서를 동료 한 명에게 공유하고, 교환 테스트처럼 피드백 받기 |
| 한 달 후 | 팀 위키에 "우리 팀 AI 지침서 v1"으로 올리기 |
