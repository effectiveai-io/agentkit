# 교환 테스트 — "남이 써도 작동하는가?"

> autoresearch에서 `program.md`의 진짜 가치는
> "나 말고 다른 사람(또는 다른 환경)에서 돌려도 같은 결과가 나오는가"입니다.
> 개인 실험에서 <strong>팀 자산</strong>으로 넘어가는 경계선이 여기입니다.

## 교환 방법 (5분)

소그룹(3~4명)에서 오른쪽 사람에게 지침서를 전달합니다.

- 지침서 v2 텍스트를 슬랙이나 메모장으로 공유
- 받은 사람은 자기 ChatGPT/Gemini 커스텀 지침에 붙여넣기

## 타인의 지침서로 AI 시키기 (15분)

받은 지침서 + 동일한 샘플 시나리오로 AI에게 시킵니다.

**관찰 포인트**:
- 내가 아닌 사람이 써도 점수가 유지되는가?
- 이해할 수 없는 약어나 암묵적 전제가 있지 않은가?
- 빠진 맥락이 있다면 무엇인가?

## 피드백 교환 (10분)

작성자에게 세 가지를 전달합니다:

| 구분 | 내용 |
|------|------|
| **Keep** | "이 줄은 확실히 효과가 있었다" |
| **Clarify** | "이건 이해가 안 됐다" |
| **Add** | "이게 빠져 있어서 결과가 달라졌다" |

<Callout type="info">
**우아한형제들도 같은 문제를 겪었습니다**

잘 쓰는 사람의 노하우가 공유는 되지만, 다른 사람이 그대로 쓸 수 없었습니다.
해결책은 노하우를 <strong>패키지로 만들어 설치</strong>하는 것이었습니다.

프롬프트, 스킬, 규칙을 하나의 npm 패키지로 관리하고,
`npm install` 한 번으로 팀 전원이 같은 수준에서 시작하게 만들었습니다.
여러분의 지침서가 바로 그 패키지의 시작입니다.

</Callout>
