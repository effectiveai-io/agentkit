# 실습: 교환 테스트

> 개인 실험에서 <strong>팀 자산</strong>으로 넘어가는 경계선.

## 교환 방법 (3분)

소그룹(3~4명)에서:
- AI가 개선한 최종 프롬프트를 옆 사람에게 공유 (슬랙/메모장)
- 받은 사람은 자기 Gemini에 붙여넣기

## 타인의 프롬프트로 테스트 (7분)

받은 프롬프트 + 동일한 샘플로 AI에게 시킵니다.

**관찰 포인트:**
- 내가 아닌 사람이 써도 점수가 유지되는가?
- 이해할 수 없는 약어나 암묵적 전제가 있지 않은가?
- 빠진 맥락이 있다면 무엇인가?

## 피드백 교환 (5분)

작성자에게 세 가지를 전달합니다:

| 구분 | 내용 |
|------|------|
| **Keep** | "이 줄은 확실히 효과가 있었다" |
| **Clarify** | "이건 이해가 안 됐다" |
| **Add** | "이게 빠져 있어서 결과가 달라졌다" |

<Callout type="info">
**우아한형제들도 같은 문제를 겪었습니다**

잘 쓰는 사람의 노하우가 공유는 되지만, 다른 사람이 그대로 쓸 수 없었습니다.
해결책은 노하우를 <strong>패키지로 만들어 설치</strong>하는 것이었습니다.
