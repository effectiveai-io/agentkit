# Phase 3 — 교환 테스트 + 축적으로 연결

## 교환 테스트 (15분)

> 개인 실험에서 <strong>팀 자산</strong>으로 넘어가는 경계선.

소그룹(3~4명)에서:

1. **교환 (3분)**: AI가 개선한 최종 프롬프트를 옆 사람에게 공유
2. **테스트 (7분)**: 받은 프롬프트를 자기 Gemini에 넣고, 같은 샘플로 테스트
3. **피드백 (5분)**:

| 구분 | 내용 |
|------|------|
| **Keep** | "이 줄은 확실히 효과가 있었다" |
| **Clarify** | "이건 이해가 안 됐다" |
| **Add** | "이게 빠져 있어서 결과가 달라졌다" |

<Callout type="info">
**우아한형제들도 같은 문제를 겪었습니다**

잘 쓰는 사람의 노하우가 공유는 되지만, 다른 사람이 그대로 쓸 수 없었습니다.
해결책은 노하우를 <strong>패키지로 만들어 설치</strong>하는 것이었습니다.

— [흩어져 있는 AI 자산, 'MCP stdio'로 헤쳐모여!](https://techblog.woowahan.com/25986/) (조현석, 2026.03)
</Callout>

## 반복에서 축적으로 (20분)

### 축적의 3레벨

```
개인 → 도구:   오늘 fork한 저장소 = 내 프롬프트의 버전 관리 시작
팀 → 자산:     팀원들이 같은 저장소를 fork하고 피드백하면 = 팀 표준
조직 → 문화:   축적된 자산 위에서 누구나 쓸 수 있으면 = 문화 전환
```

### 우아한형제들 사례

**개인 → 도구**: [테스트 자동화 플러그인](https://techblog.woowahan.com/24568/)
- 매번 같은 컨벤션 설명 반복 → 플러그인으로 구조화 → 70% 시간 단축
- "오늘 여러분이 한 것과 같습니다. 반복을 도구로 만들었습니다."

**팀 → 자산**: [MCP stdio로 헤쳐모여](https://techblog.woowahan.com/25986/)
- 흩어진 프롬프트·규칙 → npm 패키지로 중앙화
- `npm install` 한 번으로 팀 전원이 같은 수준에서 시작

**조직 → 문화**: [AI Dive Deep](https://techblog.woowahan.com/26034/)
- 마케터: 데이터 취합 150분 → 25분
- PM: MCP로 코드 정책 직접 확인
- 디자이너: Cursor AI로 피그마 플러그인 직접 제작

### 실무 적용 경로

| 오늘 | 돌아가서 |
|------|---------|
| 샘플 코드로 프롬프트 개선 | `samples/`에 **실제 팀 코드** 넣고 다시 돌리기 |
| 기본 프롬프트를 fork해서 수정 | `prompts/`를 **실제 팀 컨벤션**으로 교체 |
| Gemini에서 3사이클 돌리기 | 주 1회 돌려서 프롬프트 계속 개선하기 |
| 옆 사람과 교환 | 팀 위키에 올리고 팀 전체 피드백 |

### 도구 이름은 달라도 구조는 같다

| 도구 | 지침서가 들어가는 곳 |
|------|---------------------|
| ChatGPT | 커스텀 지침 (Settings > Customize) |
| Claude | 프로젝트 지침 (Project Instructions) |
| Claude Code / Antigravity | `CLAUDE.md` / `AGENTS.md` |
| Cursor | `.cursorrules` |
| GitHub Copilot | `copilot-instructions.md` |
| Gemini | Gems + GitHub 연결 |

## 회고 + 액션플랜

### 회고 3문항

1. autoresearch에서 가장 인상적이었던 것은?
2. AI가 자동 개선한 프롬프트에서 "이건 내가 생각 못 했다"는 순간이 있었나?
3. 월요일에 바로 할 수 있는 것 하나는?

### 액션플랜

| 타이밍 | 액션 |
|--------|------|
| 이번 주 | fork한 저장소에 실제 팀 코드/기획서를 `samples/`에 추가하고 다시 돌리기 |
| 다음 주 | 개선된 프롬프트를 동료 한 명에게 공유하고 피드백 받기 |
| 한 달 후 | 팀 저장소로 만들어서 팀 전원이 fork하고 각자 맥락에 맞게 사용하기 |
