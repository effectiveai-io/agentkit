# 워크숍 심화 — 지침서를 프로그래밍하라

<div className="workshop-narrative">

> **autoresearch에서 사람은 코드를 건드리지 않는다. 오직 `program.md`만 다듬는다.**

이 워크숍에서는 Andrej Karpathy의 <strong>autoresearch</strong>를 직접 분석하고,
같은 구조를 내 업무 프롬프트에 적용합니다.
AI가 프롬프트를 <strong>자율적으로 개선하는 루프</strong>를 돌리고,
사람은 그 방향을 조정하는 경험을 합니다.

</div>

## 3시간, 핵심 목표

<Callout type="default">
**"AI가 스스로 개선하는 루프를 설계하고, 그 방향을 조정하는 경험을 한다."**

도구 체험이 아닙니다. autoresearch의 구조를 이해하고,
같은 구조로 내 업무 프롬프트를 AI가 자동 개선하게 만듭니다.
사람은 코드 대신 <strong>지시서(program.md)</strong>를 프로그래밍합니다.
</Callout>

## 핵심 경험

1. **autoresearch를 읽고, 초등학생도 이해하게 시각화한다** — Gemini Canvas
2. **AI가 스스로 프롬프트를 개선하는 것을 지켜본다** — 자동 사이클
3. **AI의 개선 방향이 틀리면, 지시를 수정해서 다시 돌린다** — program.md 프로그래밍

## 실습 도구

| 도구 | 용도 |
|------|------|
| Gemini 웹 + GitHub 연결 | autoresearch 저장소 분석 + 자동 개선 사이클 |
| Gemini Canvas | autoresearch 개념 시각화 앱 제작 |
| GitHub 웹 | prompt-workshop 저장소 fork + 프롬프트 수정 |

## 3 Phase 흐름

<Steps>
### Phase 1: autoresearch 이해하기

<span className="step-tools">Gemini + GitHub 연결 · Gemini Canvas</span>

Gemini에 autoresearch 저장소를 연결해서 구조를 파악하고,
핵심 개념을 <strong>"초등학생도 이해할 수 있는"</strong> 시각화 앱으로 만듭니다.
옆 사람과 공유하면서 이해를 다집니다.

[Phase 1 시작하기 →](/workshop/advanced/autoresearch)

### Phase 2: 프롬프트 자동 개선 실습

<span className="step-tools">GitHub fork · Gemini + GitHub 연결</span>

[prompt-workshop](https://github.com/effectiveai-io/prompt-workshop) 저장소를 fork하고 본인 맥락에 맞게 수정합니다.
Gemini에 연결하면 AI가 `program.md`의 지시를 따라 <strong>프롬프트를 자동으로 3사이클 개선</strong>합니다.
사람은 AI의 개선 방향을 관찰하고, 필요하면 방향을 수정합니다.

[Phase 2 시작하기 →](/workshop/advanced/cycle)

### Phase 3: 교환 테스트 + 축적으로 연결

<span className="step-tools">교환 테스트 · 우형 기술블로그</span>

AI가 개선한 최종 프롬프트를 짝과 교환해서 <strong>"남이 써도 작동하는가?"</strong>를 검증합니다.
우아한형제들 기술 블로그 사례와 연결해서 실무 적용 경로를 그립니다.

[Phase 3 시작하기 →](/workshop/advanced/exchange)

</Steps>
