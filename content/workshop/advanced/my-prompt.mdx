## 내 업무 설명
나는 [직군]이고, [구체적인 반복 업무]를 하고 있어.
이 업무를 AI에게 시킬 때 쓰는 프롬프트를 자동으로 개선하는 앱이 필요해.

## 이 업무의 평가 기준
이 업무의 결과물 품질을 측정할 수 있는 평가 기준을 10개 만들어줘.
각 항목 1~5점, 만점 50점.
채점 원칙: 결과물에서 세거나 확인할 수 있는 기준으로 채점.
(예: "지적 0건 → 1점", "코드 인용 포함 → 3점" 같은 측정 가능한 기준)

## 화면 구성

### 입력 영역
- "프롬프트" 텍스트 영역
- "테스트 샘플" 텍스트 영역 (이 업무의 입력물을 넣는 곳)
- "사이클 수" 선택 (1~5, 기본값 3)
- "자동 개선 시작" 버튼

### 결과 영역
- 사이클별 총점 바 차트 (50점 만점)
- 각 사이클 상세 (아코디언):
  - 이번 프롬프트 (이전 대비 변경점 하이라이트)
  - AI가 생성한 결과물
  - 10개 항목별 점수표 + 채점 근거
  - 개선 분석
- 최종 프롬프트 복사 버튼

## 동작 방식
"자동 개선 시작"을 누르면:
1. 프롬프트로 테스트 샘플에 대한 결과물 생성
2. 평가 기준으로 채점 + 채점 근거 기록
3. 점수가 낮은 항목 분석 → 프롬프트 개선
4. 개선된 프롬프트로 반복

## 디자인
- 한국어 UI, 다크 모드
- 바 차트: 15점 이하 회색, 16~30 파란색, 31 이상 초록색

## Step 2: 내 업무 샘플 준비하기

<Timer minutes={5} />

앱이 만들어지면, 테스트할 <strong>입력물(코드, 문서, 기획서 등)을 5개</strong> 준비합니다.

### 방법 A: 실제 업무 자료가 있다면

5개를 가져옵니다.

| 직군 | 가져올 수 있는 것 예시 |
|------|---------------------|
| BE 개발 | 최근 PR의 코드 5개, API 스펙 문서 5개 |
| FE 개발 | 최근 작성한 컴포넌트 코드 5개, CSS 파일 5개 |
| PM | 최근 작성한 기획서 5개, 요구사항 문서 5개 |
| QA | 최근 테스트한 기능 설명서 5개, 결함 리포트 5개 |

### 방법 B: 지금 당장 가져오기 어렵다면

Gemini에게 내 업무와 비슷한 샘플 5개를 만들게 합니다.

```
나는 [직군]이고, [구체적인 업무 설명]을 반복적으로 하고 있어.

예를 들어 최근에 이런 일을 했어:
- [실제 업무 예시 1]
- [실제 업무 예시 2]

이 업무를 AI에게 시키기 위한 프롬프트를 테스트하려고 해.
테스트에 쓸 수 있는 현실적인 샘플을 5개 만들어줘.

조건:
- 내 업무 맥락과 비슷한 난이도와 복잡도로
- 각 샘플마다 서로 다른 유형의 문제점을 포함해줘
- 겉보기에는 괜찮아 보이지만 자세히 보면 개선할 수 있는 문제가 있는 형태로
- 너무 명백한 실수가 아니라, 도메인 지식이 있어야 발견할 수 있는 문제로
```

## Step 3: 자동 개선 사이클 돌리기

<Timer minutes={15} />

1. 내 업무에 맞는 프롬프트를 앱에 입력
2. 준비한 샘플을 테스트 영역에 붙여넣기
3. <strong>"자동 개선 시작"</strong> 클릭 → 사이클이 돌면서 점수가 올라가는 것을 확인
4. 결과를 보고 방향 조정 → 재실행
