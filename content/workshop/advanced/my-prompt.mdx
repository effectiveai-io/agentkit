## 내 업무 샘플 준비하기

본인이 AI에게 반복적으로 시키고 싶은 업무의 <strong>입력물(코드, 문서, 기획서 등)을 5개</strong> 준비합니다.

### 방법 A: 실제 업무 자료가 있다면

5개를 가져옵니다.

| 직군 | 가져올 수 있는 것 예시 |
|------|---------------------|
| BE 개발 | 최근 PR의 코드 5개, API 스펙 문서 5개 |
| FE 개발 | 최근 작성한 컴포넌트 코드 5개, CSS 파일 5개 |
| PM | 최근 작성한 기획서 5개, 요구사항 문서 5개 |
| QA | 최근 테스트한 기능 설명서 5개, 결함 리포트 5개 |

### 방법 B: 지금 당장 가져오기 어렵다면

Gemini에게 내 업무와 비슷한 샘플 5개를 만들게 합니다.

```
나는 [직군]이고, [구체적인 업무 설명]을 반복적으로 하고 있어.

예를 들어 최근에 이런 일을 했어:
- [실제 업무 예시 1]
- [실제 업무 예시 2]

이 업무를 AI에게 시키기 위한 프롬프트를 테스트하려고 해.
테스트에 쓸 수 있는 현실적인 샘플을 5개 만들어줘.

조건:
- 내 업무 맥락과 비슷한 난이도와 복잡도로
- 각 샘플마다 서로 다른 유형의 문제점을 포함해줘
- 겉보기에는 괜찮아 보이지만 자세히 보면 개선할 수 있는 문제가 있는 형태로
- 너무 명백한 실수가 아니라, 도메인 지식이 있어야 발견할 수 있는 문제로
```

---

## 자동 개선 돌리기

1. 내 업무에 맞는 프롬프트를 작성 (예: "이 기획서의 예외 케이스를 도출해줘")
2. 준비한 샘플 5개 + 내 프롬프트로 앞에서 만든 Canvas 앱에서 자동 개선 사이클 실행
3. 결과를 보고 방향 조정 → 재실행
4. 시간이 남으면 검증 앱에 5개 샘플을 넣고 범용성 검증

---

## 최종 프롬프트 저장

- 개선된 프롬프트를 메모장이나 GitHub fork에 저장
- 이것이 <strong>당장 내일부터 바로 쓸 수 있는 업무 프롬프트</strong>입니다

<Callout type="info">
**돌아가서 할 것**: 방법 B로 만든 샘플 대신 실제 업무 자료를 넣고 다시 돌리면,
지금보다 훨씬 정교한 프롬프트를 얻을 수 있습니다.
실제 자료 5개 → 10개 → 20개로 늘릴수록 프롬프트의 범용성이 올라갑니다.
</Callout>
