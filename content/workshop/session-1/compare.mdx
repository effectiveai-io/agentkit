# 세 도구 비교

**시간**: 10분 | 세 버전을 만들어본 경험으로 이야기 나눕니다.

## 오늘 경험 돌아보기

각자 아래 질문에 답해보세요. 옆사람과 이야기 나눠도 좋습니다.

1. 세 도구 중 **가장 마음에 든** 버전은 어떤 것인가요? 왜요?
2. **첫 프롬프트**와 **달각으로 개선한 프롬프트**의 결과가 어떻게 달랐나요?
3. 지금 당장 업무에 쓴다면 어떤 도구를 선택하겠나요?

## 도구 선택 기준

```
"아이디어 검증이 필요해" → Gemini Canvas
"API로 다른 시스템과 연결해야 해" → Google AI Studio
"실제로 쓸 앱이 필요해" → Lovable
```

## 오늘의 핵심

<Callout type="info" emoji="💡">
**Input 품질이 Output 품질을 결정한다**

같은 도구라도 프롬프트가 다르면 결과가 완전히 달라집니다.
달각으로 개선한 프롬프트의 결과를 직접 눈으로 봤죠.

이게 세션 3에서 "프롬프트 자산화"를 하는 이유입니다.
좋은 프롬프트를 한 번 만들면, 계속 써야 하니까요.
</Callout>
