# Step 2: Google AI Studio

**시간**: 15분 | **도구**: [Google AI Studio](https://aistudio.google.com)

## 목표

System Prompt 개념을 이해하고, Gemini API를 직접 연결하는 방식으로 앱을 구성한다.

## 실습

<Steps>
### Google AI Studio 접속

[aistudio.google.com](https://aistudio.google.com) → `Create new prompt` 클릭

### System Prompt 작성 (7분)

AI Studio에서 핵심은 **System Prompt** — AI가 항상 따르는 역할 정의입니다.

왼쪽 `System instructions` 영역에 아래를 채워보세요:

```
당신은 프롬프트 개선 전문가입니다.

사용자가 프롬프트를 입력하면:
1. [여기에 AI가 해야 할 일을 작성하세요]
2. [출력 형식을 지정하세요]
```

### 모델 설정 확인 (3분)

오른쪽 패널에서:
- **Model**: `gemini-2.0-flash` 선택
- **Temperature**: `0.7` (창의성과 일관성의 균형)
- **Max output tokens**: `1024`

### 테스트 & 비교 (5분)

Step 1에서 만든 Gemini Canvas 버전과 결과를 비교해보세요.

- System Prompt가 있으니 어떤 점이 다른가요?
- 대화가 이어질수록 더 일관성 있게 동작하나요?
</Steps>

## AI Studio의 특징

| 장점 | 단점 |
|------|------|
| System Prompt로 역할 정의 가능 | UI 만들기는 별도 작업 필요 |
| API 키로 다른 앱에 연결 가능 | 비개발자에겐 복잡해 보일 수 있음 |
| 갤러리의 다양한 예제 참고 가능 | 배포 불가 (API로만 활용) |

<Callout type="info">
**System Prompt = CLAUDE.md**

AI Studio의 System Prompt는 `CLAUDE.md`와 같은 개념입니다.
"AI가 항상 이런 방식으로 동작하도록" 한 번 설정해두는 것이죠.
</Callout>
