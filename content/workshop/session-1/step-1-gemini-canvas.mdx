# Step 1: Gemini Canvas

**시간**: 15분 | **도구**: [Gemini](https://gemini.google.com) → Canvas 모드

## 목표

코드 한 줄 없이, 프롬프트 하나로 작동하는 프롬프트 증강기를 만든다.

## 실습

<Steps>
### Gemini 접속 후 Canvas 모드 활성화

[gemini.google.com](https://gemini.google.com) 접속 → 입력창 오른쪽 `Canvas` 아이콘 클릭

### 첫 번째 프롬프트 작성 (5분)

각자 아래 빈칸을 채워서 프롬프트를 작성하고 실행해보세요.

```
[여기에 내가 만들고 싶은 프롬프트 증강기를 설명하세요]
```

결과를 보고: 원하는 모습이 나왔나요? 어떤 부분이 부족한가요?

### 딸깍 참고 후 프롬프트 개선 (5분)

[딸깍(dalgak.com)](https://dalgak.com)을 열고, 내가 쓴 프롬프트를 딸깍에 넣어보세요.
딸깍이 제안하는 개선 프롬프트로 Gemini Canvas에 다시 시도해보세요.

**비교 포인트**: 첫 번째 결과 vs 개선된 프롬프트의 결과, 무엇이 달라졌나요?

### 완성 버전 확인 (5분)

강사의 프롬프트와 비교해봅니다.

<Tabs items={['강사 프롬프트 (공개 전)', '힌트']}>
  <Tabs.Tab>
    실습 후 공개 예정
  </Tabs.Tab>
  <Tabs.Tab>
    - 앱의 **목적**을 명확히 설명했나요?
    - 입력과 출력 형식을 **구체적으로** 지정했나요?
    - 사용자가 어떻게 **상호작용**하는지 설명했나요?
  </Tabs.Tab>
</Tabs>
</Steps>

## Gemini Canvas의 특징

| 장점 | 단점 |
|------|------|
| 설정 없이 즉시 시작 | URL 공유 불가 |
| 대화하며 실시간 수정 | Google 계정에 묶임 |
| 코드 편집도 가능 | 배포 불가 |

"이런 게 되는지" 5분 안에 확인하고 싶을 때 사용하세요.
</Callout>
