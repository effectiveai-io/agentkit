# 3-2. Before / After 비교하기

AGENTS.md가 실제로 차이를 만드는지 실험합니다.

## Before (Knowledge 없음)

<Steps>
### 빈 프로젝트 만들기

Lovable에서 새 프로젝트를 만듭니다.

```
빈 프로젝트 하나 만들어줘. 흰 화면만 있으면 돼
```

### 프롬프트 증강기 요청하기

프로젝트가 생성되면 아래 프롬프트를 입력하세요.

```
프롬프트 증강기 만들어줘
```

### 결과 저장하기

결과를 스크린샷 찍거나 저장해 주세요.
</Steps>

---

## After (Knowledge 있음)

<Steps>
### Knowledge 설정하기

같은 프로젝트에서 설정 메뉴 → **Knowledge**를 클릭합니다.

### AGENTS.md 붙여넣기

방금 작성한 AGENTS.md 내용을 전부 붙여넣습니다.

### 다시 만들기

채팅에 아래 프롬프트를 입력하세요.

```
지금까지 만든 것 무시하고, 처음부터 다시 만들어줘. 프롬프트 증강기 만들어줘.
```

### 비교하기

Before와 After 결과를 비교해보세요.
</Steps>

---

## 비교 체크리스트

- [ ] 디자인이 내가 원하는 스타일(노션풍 미니멀 등)로 나왔는가?
- [ ] 모바일 반응형이 적용되었나?
- [ ] 로딩 처리(스켈레톤 UI)가 있는가?
- [ ] 결과 표시 방식이 AGENTS.md에 적은 대로인가? (인라인 vs 팝업 등)
- [ ] 한국어 UI가 기본 적용되었나?

---

## 후속 요청 테스트

추가로 후속 요청도 테스트해보세요.

```
다크 모드 추가해줘
```

Knowledge 규칙이 후속 요청에서도 유지되는지 확인합니다.

<Callout type="info" emoji="🔑">
**핵심 차이**: 프롬프트에 직접 규칙을 붙여넣으면 그 한 번만 적용됩니다.
Knowledge에 넣으면 기능 추가, 버그 수정, 리팩토링 등 **모든 후속 대화에 자동 적용**됩니다.

이것이 AGENTS.md의 본질 — **"한 번 설명하면 매번 적용된다"**.
</Callout>
