# 3-1. AGENTS.md 초안 작성하기

방금 적은 "반복한 말"을 AGENTS.md 형식으로 정리합니다.
아래 템플릿을 복사해서 시작하세요.

## 템플릿

```markdown
# AGENTS.md

## 프로젝트 개요
(이 프로젝트/팀이 뭘 하는지 한 줄로)

## 코딩 규칙
(반복했던 기술 관련 요청을 규칙으로)
- 예: 모든 UI는 모바일 반응형으로 구현한다
- 예: 로딩 상태는 스켈레톤 UI로 표시한다

## 디자인 원칙
(반복했던 디자인 관련 요청을 원칙으로)
- 예: 노션 스타일의 미니멀한 디자인을 따른다
- 예: 색상은 [브랜드 컬러]를 사용한다

## 커뮤니케이션
(반복했던 소통 관련 요청을 규칙으로)
- 예: 한국어로 응답한다
- 예: 코드에 한국어 주석을 단다

## 하지 말 것
(AI가 자주 하지만 내가 원하지 않는 것)
- 예: 불필요한 애니메이션 추가하지 않기
- 예: 설명 없이 코드만 던지지 않기
```

## 프롬프트 증강기 프로젝트 예시

아래는 오늘 만든 프롬프트 증강기에 맞는 AGENTS.md 예시입니다. 참고해서 본인 버전을 작성해보세요.

```markdown
# AGENTS.md

## 프로젝트 개요
사용자의 프롬프트를 더 구체적이고 효과적으로 개선해주는 웹앱

## 코딩 규칙
- 모든 페이지는 모바일 반응형으로 구현한다 (min-width: 375px)
- 데이터 로딩 중에는 스켈레톤 UI를 표시한다
- 버튼 클릭 시 결과를 즉시 보여준다 (로딩 스피너 아님, 스켈레톤)

## 디자인 원칙
- 노션 스타일의 미니멀한 디자인을 따른다
- 배경색은 흰색, 카드는 연한 그레이 보더
- 폰트는 Pretendard를 사용한다

## 커뮤니케이션
- 모든 UI 텍스트는 한국어로 작성한다
- 에러 메시지는 사용자가 다음에 뭘 해야 하는지 안내한다

## 하지 말 것
- 불필요한 페이드인/아웃 애니메이션 넣지 않기
- 결과를 팝업/모달로 보여주지 않기 (인라인으로)
```

## 작성 기준

[Claude Code 공식문서](https://code.claude.com/docs/en/memory#write-effective-instructions) 기반:

- **넣어야 하는 것**: AI가 코드를 읽어도 알 수 없는 것, 우리 팀만의 방식
- **넣지 말 것**: "클린 코드를 작성하라" 같은 자명한 것
- **구체적이고 검증 가능하게**: "모바일 반응형으로 구현한다" ✓ / "UI를 잘 만들어라" ✗
- **분량**: ~200줄 이하 ("Files over 200 lines consume more context and may reduce adherence")
