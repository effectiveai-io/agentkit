## 2-2. Lovable에서 프롬프트 증강기 v3 만들기

<Steps>
### Lovable 접속하기

[lovable.dev](https://lovable.dev)에 접속합니다.

### 워크스페이스 확인하기

**빗썸 실습용 워크스페이스**에서 작업하고 있는지 확인하세요.

### 프롬프트 입력하기

아래 프롬프트를 입력하세요.

```
프롬프트 증강기 웹앱을 만들어줘.

기능:
- 사용자가 프롬프트를 입력하면 AI가 더 효과적인 프롬프트로 개선
- 짧은 버전 / 중간 버전 / 상세 버전 3가지를 탭으로 보여줘
- 각 버전마다 복사 버튼
- 사용 이력을 저장해서 이전에 개선한 프롬프트를 다시 볼 수 있게

디자인:
- 노션 스타일의 미니멀한 UI
- 모바일 반응형
- 로딩 중일 때 스켈레톤 UI 표시
```

### 수정 요청하기

첫 버전이 나오면 **최소 1가지는 본인이 직접 수정 요청**해보세요.

<Callout type="warning" emoji="✏️">
**수정 요청 예시:**
- "그라데이션 컬러 배경을 넣어줘"
- "다크 모드 토글 추가해줘"
- "결과를 마크다운으로 렌더링해줘"
</Callout>
</Steps>

## 2-3. Google 인증 + DB 연동하기

Lovable Cloud 기능으로 로그인과 데이터 저장을 붙여봅니다.

<Steps>
### 프롬프트 입력하기

채팅에 아래 프롬프트를 입력하세요.

```
아래 기능을 추가해줘:

1. Google 로그인: 로그인한 사람만 증강기를 사용할 수 있게 해줘.
   로그인하지 않은 사람에게는 로그인 버튼만 보여줘.

2. 프롬프트 저장: 개선된 프롬프트를 "저장" 버튼으로 DB에 저장할 수 있게 해줘.
   로그인한 유저별로 데이터가 분리되어야 해.
   "내 저장 목록" 페이지에서 이전에 저장한 프롬프트를 다시 볼 수 있게 해줘.
```

### Supabase 연동하기

Lovable이 Supabase 연동을 안내하면 따라서 진행하세요.

### 테스트하기

실제로 Google 계정으로 로그인이 되는지 테스트합니다.

### 저장 기능 확인하기

프롬프트를 저장하고, "내 저장 목록"에서 확인해보세요.
</Steps>

<Callout type="info" emoji="💡">
**핵심 체감 포인트:** Canvas나 AI Studio에서는 별도로 해야 했던 인증·DB가
Lovable에서는 대화 한 번으로 끝납니다.
</Callout>

## 2-4. 배포 및 공유

<Steps>
### 배포하기

우상단 **Publish** 버튼을 눌러 배포합니다 — 이것만으로 누구나 접속 가능한 URL이 생깁니다.

### URL 공유하기

생성된 URL을 시트에 붙여넣어주세요.

### 다른 사람 앱 써보기

옆 사람의 URL을 클릭해서 로그인 → 프롬프트 저장까지 직접 써보세요.
</Steps>
