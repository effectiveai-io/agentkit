## 몬테카를로 시뮬레이션이란?

**몬테카를로(Monte Carlo)** — 모나코의 유명한 카지노 도시 이름에서 따왔습니다.
1940년대 맨해튼 프로젝트 과학자들이 핵반응을 예측하기 위해 개발하며 붙인 이름이에요.

> "수학 공식으로 못 풀겠으니, 그냥 주사위 엄청 많이 던져보자"

**핵심 아이디어:** 복잡한 수학 공식 대신, 무작위 시행을 수천~수만 번 반복해서 결과의 분포를 직접 확인한다.

**비유로 이해하기** — 주사위 두 개를 던졌을 때 합이 7이 나올 확률은?

- **수학적 풀이**: 경우의 수 계산… 6/36 = 16.67%
- **몬테카를로**: 10,000번 던져보니 약 1,667번 나옴 → 16.67%

결과는 같은데, 몬테카를로는 **복잡한 상황에서도 작동**합니다!

| 분야 | 활용 사례 |
|------|----------|
| 금융 | 포트폴리오 리스크 분석, 옵션 가격 산정 |
| 프로젝트 관리 | 일정 추정, 리스크 평가 |
| 게임/AI | 바둑 AI (알파고), 체스 엔진 |
| 날씨 | 기상 예보, 허리케인 경로 예측 |

## 실습 1: 가짜 동전 던지기 감별법

### 상황 설정

그런데 귀찮아서 그냥 지어서 제출한 학생들이 있습니다. 교수는 어떻게 가짜를 걸러낼 수 있을까요?

**인간은 "랜덤"을 제대로 못 만듭니다:**
- 가짜로 적은 학생 → "앞앞앞앞앞" 같은 연속을 **일부러 피함**

## 핵심 개념: 삼각분포 (min / mod / max)

여기까지의 실습은 확률이 정해져 있었습니다 (동전 50%, 주사위 각 면 1/6, 생일 1/365). 하지만 현실의 불확실성은 다릅니다.

> "출근 시간이 얼마나 걸릴까?" "이 작업이 며칠 걸릴까?"
> → 정확한 확률이 없다!

<strong>삼각분포</strong>는 불확실한 값을 추정할 때 가장 직관적인 방법입니다. <strong>세 가지 값만 알면 됩니다:</strong>

| 값 | 의미 | 예시 (출근 시간) |
|----|------|----------------|
| **min** | 최소값 (가장 빨리) | 25분 |
| **mod** | 최빈값 (가장 자주) | 35분 |
| **max** | 최대값 (가장 늦게) | 50분 |

**왜 삼각분포가 유용한가:**
- 전문 지식 없이도 "최소/보통/최대"는 누구나 추정 가능
- 동료에게 물어보기 쉬움: "이 작업 빠르면? 보통? 늦으면?"
- PERT(프로젝트 관리), 리스크 분석의 표준 기법

## 실습 4: 출근 시간 계산기

### 상황 설정

매일 아침 반복되는 고민… "9시까지 출근인데, 몇 시에 나가야 하지?"

빠르면 25분, 보통 35분, 막히면 50분. "보통 35분이니까 8시 25분에 나가면 되겠지?" → **이러면 절반 확률로 지각합니다.**

질문을 바꿔봅시다:
- ❌ "보통 얼마나 걸려?" → 50% 확률로 지각
- ✅ "90% 확률로 제시간에 도착하려면?" → 한 달에 2번 정도만 아슬아슬

<Steps>
### 프롬프트 입력하기

```
출근 시간 계산기 웹앱을 만들어줘.

## 배경

매일 아침 "몇 시에 나가야 하지?" 고민하는데,
"보통 35분 걸리니까 8시 25분에 나가면 되겠지"라고 생각하면
실제로는 절반 확률로 지각하게 돼.
불확실성을 고려해서 "N% 확률로 제시간 도착하는 출발 시간"을 계산하고 싶어.

## 기능

1. 입력값
   - 출근 소요시간: min/mod/max (기본값: 25분/35분/50분)
   - 목표 도착 시간 (기본값: 오전 9시)
   - 시뮬레이션 반복 횟수 슬라이더
     (기본값: 10,000, 최대값: 1,000,000, 최소값: 1,000)

2. 출력값
   - 소요시간 분포 히스토그램
   - 확률별 권장 출발 시간 표:
     - 50% (반반) → 8시 ??분
     - 80% (꽤 안전) → 8시 ??분
     - 90% (안전) → 8시 ??분
     - 95% (매우 안전) → 8시 ??분
   - 각 백분위수를 히스토그램에 세로선으로 표시
   - "90% 안전 출발 시간"을 크게 강조

3. 추가 기능
   - 출발 시간 입력하면 → 지각 확률 계산

4. UI
   - 슬라이더로 min/mod/max 조절 가능
   - 도착 시간도 선택 가능 (9시, 9시 30분 등)
   - 결과가 직관적으로 보이게

## 핵심 인사이트

"보통 35분이니까 8시 25분에 출발하면" → 50% 지각
"90% 안전하려면 8시 N분에 출발하기"
이런 차이를 시각적으로 명확하게 보여줘.
```

<Callout emoji="👀">
**완성된 예시를 먼저 보고 싶다면:** [출근 시간 계산기 데모](https://gemini.google.com/share/24176d0f2ddb)
</Callout>

### 결과 확인하기

"보통" 기준 출발 시간과 "90% 안전" 출발 시간의 차이를 확인해보세요.
</Steps>

## 실습 5: 업무 일정 예측기

PM: "이 프로젝트 언제 끝나요?"

과연 30일 안에 끝날 확률은 몇 %일까요?

| 작업 | min | mod | max |
|------|-----|-----|-----|
| 기획 | 5일 | 7일 | 12일 |
| 디자인 | 3일 | 5일 | 8일 |
| 개발 | 10일 | 14일 | 25일 |

<Steps>
### 프롬프트 입력하기

```
업무 일정 예측기 웹앱을 만들어줘.

## 배경

PM이 "이 프로젝트 언제 끝나요?"라고 물으면 항상 난감해.
각 작업마다 빠르면/보통/늦으면 소요시간이 다른데,
단순히 중간값 더해서 대답하면 거의 항상 틀리거든.
불확실성을 고려해서 "N일 안에 끝날 확률"을 계산하고 싶어.

## 기능

1. 입력값
   - 작업 목록 (동적으로 추가/삭제 가능)

   - 시뮬레이션 반복 횟수 슬라이더

   - 기본 예시 3개 세팅:
     기획(5/7/12), 디자인(3/5/8), 개발(10/14/25)

2. 출력값
   - 전체 일정의 평균, 중간값
   - 주요 확률 구간:
     25일/30일/35일/40일 안에 끝날 확률
   - 50%, 80%, 90%, 95% 확률로 끝나는 일수
   - 전체 일정 분포 히스토그램
   - 각 백분위수를 히스토그램에 세로선으로 표시

3. UI
   - 작업 추가/삭제 버튼
   - 각 작업별 min/mod/max 입력 필드
   - "시뮬레이션 실행" 버튼
   - 결과는 표 + 히스토그램으로 시각화
   - "90% 확률 일정"을 강조해서 보여주기 (협상용)

## 핵심 인사이트

"중간값 합산으로만 약속하면 50% 확률로 못 지킨다"는 걸 보여주고,
"90% 확률로 지키려면 실제로 며칠을 잡아야 하는지" 알려줘.
협상할 때 근거 자료로 쓸 수 있게.
```

<Callout emoji="👀">
**완성된 예시를 먼저 보고 싶다면:** [업무 일정 예측기 데모](https://gemini.google.com/share/c09b07f0772a)
</Callout>

### 결과 확인하기

중간값 합산 26일과 "90% 확률 일정"의 차이가 얼마나 되는지 확인해보세요.

</Steps>

<Callout emoji="💡">
**인사이트:** "각각 중간값 더하면 26일이니까 30일이면 충분하죠" → 실제로는 약 55% 확률로만 가능합니다. 불확실성은 합쳐질수록 커집니다. 일정 협상할 때 이런 데이터가 있으면 설득력이 올라갑니다.
</Callout>

## 실습 6: 매출/예산 시나리오 분석

### 상황 설정

경영진: "이번 분기 매출 목표 1억 달성 가능해요?"
팀장: "네, 가능할 것 같습니다" (확신 없음)

**"아마도요" 말고 확률로 대답할 수 없을까요?**

| 월 | 보수적 | 예상 | 낙관적 |
|----|--------|------|--------|
| 1월 | 2,500만 | 3,000만 | 3,500만 |
| 2월 | 2,800만 | 3,200만 | 4,000만 |
| 3월 | 3,000만 | 3,500만 | 4,500만 |

질문을 바꿔봅시다:
- ❌ "1억 될 것 같습니다" → 근거 없음
- ✅ "1억 달성 확률은 42%입니다" → 객관적 근거
- ✅ "80% 확률로 9,200만 이상 달성합니다" → 보수적 목표 제시

<Steps>
### 프롬프트 입력하기

```
매출/예산 시나리오 분석기 웹앱을 만들어줘.

## 배경

경영진이 "이번 분기 목표 달성 가능해?"라고 물으면
"아마도요"라고 대답하기 싫어.
월별 매출 추정치를 입력하면 분기 전체의 확률 분포를 계산해서
"목표 달성 확률 N%"로 보고하고 싶어.

## 기능

1. 입력값
   - 기간 항목 (동적으로 추가/삭제 가능, 기본 3개월)
   - 각 기간별 min/mod/max 매출 (삼각분포)
   - 목표 금액 입력 (기본값: 1억)
   - 시뮬레이션 반복 횟수
     (기본값: 10,000, 최대값 100,000, 최소값 1,000)
   - 기본 예시:
     - 1월(2500/3000/3500만)
     - 2월(2800/3200/4000만)
     - 3월(3000/3500/4500만)

2. 출력값
   - 합계의 평균, 중간값
   - 목표 금액 달성 확률 (크게 강조!)
   - 시나리오별 매출:
     - 최악 (5%): ?만원
     - 보수적 (25%): ?만원
     - 중간 (50%): ?만원
     - 낙관적 (75%): ?만원
     - 최상 (95%): ?만원
   - 매출 분포 히스토그램
   - 목표 금액을 히스토그램에 세로선으로 표시

3. UI
   - 기간 추가/삭제 버튼
   - 금액 입력은 "만원" 단위로 (가독성)
   - 목표 달성 확률을 가장 크게 표시
   - 경영진 보고용으로 깔끔하게

## 핵심 인사이트

"예상 합계 N이니까 목표액 M억은 힘들 것 같아요"가 아니라
"목표액 M억 달성 확률은 X%입니다. Y% 확률로는 N 이상입니다"
이런식으로 확률 기반으로 보고할 수 있게 해줘.
```

<Callout emoji="👀">
**완성된 예시를 먼저 보고 싶다면:** [매출 시나리오 분석기 데모](https://gemini.google.com/share/0578d07d902e)
</Callout>

### 결과 확인하기

목표 금액을 바꿔가며 달성 확률이 어떻게 변하는지 확인해보세요.
</Steps>
