# 세션 3: 전문성 자산화

**시간**: 60분 | **목표**: 내 전문성을 AI가 재현 가능한 팀 자산으로 만든다

## 지금까지 만든 것

```
세션 1: 프롬프트 증강기 첫 버전 (Gemini Canvas → AI Studio → Lovable)
세션 2: Google 인증 + DB 저장 + AI 연동 → 실제 동작하는 앱
세션 3: 나만 아는 노하우를 팀 모두가 쓸 수 있는 자산으로
```

지금은 **나만 쓸 수 있는 앱**입니다. 로그인한 내 계정의 프롬프트만 볼 수 있죠.

이번 세션에서는 이걸 **팀이 함께 쓰는 지적 자산**으로 바꿉니다.

<Callout type="info">
**왜 자산화인가요?**

에이전틱 코딩의 핵심은 "내 판단을 언어로 만드는 것"입니다.
그 판단(프롬프트)을 팀이 공유하면 개인의 노하우가 조직의 경쟁력이 됩니다.

> "프롬프트를 자산화한다는 것 = 나의 전문성을 AI가 재현 가능하게 만드는 것"

달각(dalgak.com), Claude 플러그인들 — 이것들이 모두 같은 관점에서 만들어진 도구들입니다.
</Callout>
