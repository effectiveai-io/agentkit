# 팀 공유 기능 추가

**시간**: 25분

## 목표

저장된 프롬프트에 "공개/비공개" 옵션을 추가하고, 공개된 프롬프트는 팀 전체가 볼 수 있도록 만든다.

## 실습

<Steps>
### 공개/비공개 필드 추가

<Tabs items={['도전해보기', '힌트 프롬프트']}>
  <Tabs.Tab>
    프롬프트에 공개/비공개 옵션을 추가하는 프롬프트를 써보세요.
    DB 스키마 변경, UI 변경, 권한 처리를 모두 고려해야 합니다.
  </Tabs.Tab>
  <Tabs.Tab>
    ```
    프롬프트 공유 기능을 추가해줘.

    ## DB 변경
    - prompts 테이블에 is_public (boolean, default false) 컬럼 추가
    - is_public이 true이면 모든 사용자가 읽기 가능하도록 RLS 정책 추가

    ## UI 변경
    - 저장 시 "팀에 공개" 토글 추가
    - 내 프롬프트 목록에서 공개/비공개 상태 표시 아이콘
    - 공개 여부를 언제든 변경 가능

    ## 팀 프롬프트 탭
    - "팀 프롬프트" 탭 추가
    - 모든 사용자의 공개 프롬프트를 최신순으로 표시
    - 누가 공유했는지 이름/아바타 표시
    - 팀 프롬프트를 내 앱에 바로 적용 가능한 '사용하기' 버튼
    ```
  </Tabs.Tab>
</Tabs>

### 팀원과 테스트

강사가 공유한 앱 URL로 모두 함께 접속합니다.

각자:
1. 세션 2에서 만든 프롬프트 중 하나를 공개로 전환
2. "팀 프롬프트" 탭에서 팀원들의 공개 프롬프트 확인
3. 마음에 드는 프롬프트로 '사용하기' 클릭

<Callout>
지금 화면에 팀원들의 프롬프트가 실시간으로 쌓이고 있습니다.
이게 **팀 자산화**의 시작입니다.
</Callout>
</Steps>
