# AI 엔지니어 커리큘럼 — CodePath × Anthropic

<Callout type="info">
원문: [Anthropic × CodePath 커리큘럼 리뷰](https://flowkater.io/posts/2026-03-06-anthropic-codepath-curriculum/) — flowkater
</Callout>

Claude를 만든 Anthropic이 가르치는 것은 역설적으로 **"AI 코드를 의심하는 법"**입니다.
CodePath와의 파트너십으로 만든 30주+ 커리큘럼은 "AI로 빨리 만드는 법"이 아니라,
<strong>AI와 함께 일하면서도 기초를 잃지 않는 엔지니어</strong>를 키우는 것을 목표로 합니다.

---

## 커리큘럼 구성 (3단계, 30주+)

<Steps>

### 1단계: AI 엔지니어링 기초 (10주)

1주차부터 과제가 "AI가 짠 코드에서 버그를 찾아라"입니다.

- AI가 생성한 5개 정렬 알고리즘 중 틀린 것 찾기
- 프롬프트 3가지로 같은 문제를 풀고 결과 비교 분석
- RAG, 에이전트 워크플로우, 가드레일 구현 실습
- 10주차에 챗봇·요약 도구 같은 실제 작동하는 결과물 제출

핵심은 **"생산에서 검증으로"** 사고를 전환하는 것입니다.

### 2단계: AI 엔지니어링 응용 (10주)

실전입니다. 1만 줄 이상의 기존 코드베이스에 AI를 끼워 넣습니다.

- 1만 줄+ 오픈소스 코드베이스를 받아 구조 파악
- AI가 못 풀어서 직접 디버깅한 부분을 기록 (학습 증명)
- 다른 팀의 AI PR을 리뷰하며 검증 기준 체크리스트 작성
- 실제 오픈소스 프로젝트에 PR 제출, 메인테이너 리뷰 받기

핵심은 **"기존 코드에 끼워 넣기"** — AI는 신규 개발을 잘하지만, 통합은 사람이 해야 합니다.

### 3단계: 오픈소스 캡스톤 (지속형)

포트폴리오에 넣을 수 있는 **실제 오픈소스 기여**가 졸업 요건입니다.

</Steps>

## AI Native Engineer의 4가지 조건

Anthropic이 정의하는 AI 시대 엔지니어의 인재상입니다:

| 조건 | 설명 |
|------|------|
| **비판적 코드 평가** | AI의 1%짜리 오류를 찾아내는 능력 |
| **대규모 코드베이스 이해** | 코드 읽기 능력의 가치가 오히려 상승 |
| **프로덕션 레벨 관심사** | 에러 핸들링, 보안, 성능, 모니터링까지 책임 |
| **실전 기여 능력** | 실제 메인테이너의 검수를 받은 경험 |

## 핵심 인사이트

### 기초 없이는 AI가 무게가 된다

Carnegie Mellon 교수 Po-Shen Loh의 비유:

> "AI로 글쓰기를 대신하는 건 운동 대신 차를 타는 것 — 목적지에 도착하지만 체력은 안 생긴다."

자료구조를 모르면 AI가 만든 알고리즘의 버그를 찾을 수 없습니다.
진입 장벽은 낮아졌지만, **천장은 오히려 높아졌습니다.**

### 병목의 이동

Andrew Ng이 강조한 포인트:

> 이제 병목은 "코드 작성"에서 **"무엇을 만들지 결정"**으로 옮겨갔다.

PM과 엔지니어의 역할이 합쳐지는 추세입니다.
빠르게 만들 수 있게 되었기 때문에, "무엇을 안 만들지"가 더 중요해졌습니다.

### 멘토링 현장에서 본 차이

165개 학생 AI 대화록을 분석한 결과:

- **성장 부진**: "코드 짜줘" → 복사 → 생각 없음
- **빠른 성장**: 자기 가설 수립 → AI 검증 요청 → 통합

## 워크숍과의 연결

이 커리큘럼의 철학은 우리 워크숍의 구조와 정확히 맞닿아 있습니다:

- **세션 1 (만든다)** = 1단계의 "일단 AI로 만들어보기"
- **세션 2 (반복 발견)** = "AI가 못 풀어서 직접 디버깅한 부분 기록"
- **세션 3 (자산화)** = "검증 기준 체크리스트 → AGENTS.md로 자산화"

차이점이 있다면, CodePath는 30주에 걸쳐 체계적으로 진행하고,
우리 워크숍은 3세션에 핵심 사이클을 압축해서 **패턴을 먼저 체험**하게 합니다.
