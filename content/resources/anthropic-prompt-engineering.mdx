# 프롬프트 엔지니어링 가이드

<Callout type="info">
공식 문서 (한국어): [platform.claude.com/docs/ko — 프롬프트 엔지니어링](https://platform.claude.com/docs/ko/build-with-claude/prompt-engineering/overview)
</Callout>

Anthropic이 공개한 프롬프트 엔지니어링 공식 가이드입니다.
"프롬프트를 잘 쓰면 파인튜닝 없이도 성능을 비약적으로 끌어올릴 수 있다"는 것이 핵심 메시지입니다.

---

## 시작하기 전에

Anthropic은 프롬프트 엔지니어링을 시작하기 전에 세 가지를 먼저 갖추라고 강조합니다:

1. **성공 기준의 명확한 정의** — "잘 되었다"를 어떻게 판단할 것인가
2. **경험적 테스트 방법** — 결과를 반복 측정할 수 있는 구조
3. **개선할 초안 프롬프트** — 출발점이 있어야 개선이 가능

---

## 8가지 핵심 기법

Anthropic은 가장 광범위하게 효과적인 기법부터 전문적인 기법 순으로 학습하길 권장합니다:

<Steps>

### 프롬프트 생성기

Claude Console에서 제공하는 도구로, 초안 프롬프트가 없을 때 시작점을 만들어줍니다.

### 명확하고 직접적으로 작성하기

모호한 표현 대신 구체적인 지시를 사용합니다. Claude는 문자 그대로 따르므로, 원하는 것을 정확히 말할수록 좋습니다.

### 예시 사용하기 (멀티샷 프롬프팅)

원하는 출력 형식과 스타일을 예시로 보여줍니다. 2~3개의 예시만으로도 품질이 크게 향상됩니다.

### Claude에게 생각하게 하기 (사고의 연쇄)

"단계별로 생각해보세요"와 같은 지시로 Claude가 추론 과정을 거치게 합니다. 복잡한 문제에서 정확도가 높아집니다.

### XML 태그 사용하기

`<instructions>`, `<context>`, `<output>` 같은 XML 태그로 프롬프트를 구조화합니다. Claude는 XML 태그에 잘 반응합니다.

### Claude에게 역할 부여하기 (시스템 프롬프트)

시스템 프롬프트로 Claude의 페르소나와 행동 규칙을 설정합니다. 일관된 응답 스타일을 유지하는 데 효과적입니다.

### 복잡한 프롬프트 체이닝

하나의 큰 작업을 여러 단계로 나누고, 각 단계의 출력을 다음 단계의 입력으로 연결합니다.

### 긴 컨텍스트 팁

대량의 문서를 다룰 때 Claude가 핵심 정보를 놓치지 않도록 하는 기법입니다.

</Steps>

---

## 왜 프롬프트 엔지니어링인가?

Anthropic은 파인튜닝 대신 프롬프트 엔지니어링을 우선 권장하는 이유를 명확히 합니다:

| 프롬프트 엔지니어링 | 파인튜닝 |
|-------------------|---------|
| 텍스트 입력만 필요 | GPU와 대용량 메모리 필요 |
| 즉각적인 결과 확인 | 수 시간~수 일 소요 |
| 모델 업데이트 시 그대로 작동 | 재학습 필요 |
| 모델의 일반 지식 보존 | 치명적 망각 위험 |
| 투명하고 디버깅 용이 | 블랙박스 |

## 워크숍과의 연결

이 워크숍에서 **프롬프트 증강기**를 만드는 과정이 바로 프롬프트 엔지니어링의 실전 적용입니다.

- **세션 1**에서 다양한 프롬프트를 시도하며 "멀티샷"과 "명확한 작성"을 체험
- **세션 2**에서 반복 패턴을 발견하며 "프롬프트 체이닝"을 자연스럽게 학습
- **세션 3**에서 AGENTS.md로 자산화하며 "시스템 프롬프트"와 "XML 태그"를 활용
